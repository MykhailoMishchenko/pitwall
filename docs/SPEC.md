# Pitwall — спецификация

Платформа аналитики Формулы-1 + игра в официальные F1 Fantasy и F1 Predict.
Наследник Gaffer (wc2026-analytics): Claude-движок, read-only витрина,
самообучение через цикл «прогноз → результат → разбор → learnings».

## Три контура

1. **Локальное зеркало (source of truth)** — аналитика чемпионата, команд,
   пилотов, стратегий; журнал пиков; разборы; learnings.
2. **Дублирование на официальную платформу** — каждый состав фэнтези и ответы
   Predict уходят на реальный аккаунт пользователя (детерминированные
   сабмиттеры после Phase 0; фолбэк — пользователь вносит руками из отчёта).
3. **Лига против людей** — приватные лиги Fantasy + Predict; участник
   «PITWALL by Claude»; соперники — знакомые пользователя (в т.ч. Макс
   Подзигун). Вход с R10/24, все стартуют с нуля — честно.

## Цикл уикенда (полная автономия)

Замыкающая рутина разбора читает календарь следующего ГП (Jolpica) и сама
сетапит рутины под его времена. Ничего не хардкодится: времена сессий у
каждого ГП свои; спринт-формат сдвигает всё раньше (лок = спринт-квала Пт)
и включает двойной скоринг.

| Когда | Рутина |
|---|---|
| Вт–Ср | Превью: профиль трассы, форма, цены, погода → черновик пиков v0 |
| Пт | Уточнение: практики (FastF1 лонг-раны, деградация) → пики v1 |
| Сб, лок −3ч | Финальные пики: журнал (таймстемп) → отправка → верификация |
| Сб, лок −1ч | Страховка: не подтверждено → пуш пользователю |
| Сб, после квалы | Квала-разбор; окно boost-корректировок Predict |
| Вс, после гонки | Разбор полётов: скоринг, прогноз vs факт, learnings, лига |
| Вс–Пн | Замыкание: календарь следующего ГП → сетап рутин → отчёт |

## Доктрина игры (решение Claude, мандат пользователя: не сухая математика)

1. **Лига-осознанный риск** — отстаём: differential-пики и дисперсия;
   лидируем: кроемся популярными пиками.
2. **Conviction-пики** — право переиграть модель есть; каждое отклонение
   логируется ДО лока с обоснованием.
3. **Тіньова команда** — чистая модель без вольностей весь сезон параллельно;
   финальный ответ «мнение добавляет или сжигает очки».
4. **Сезонное планирование** — чипы/boost'ы под календарь (все 6 чипов на
   15 уикендов); спекуляция ценами (покупка до подорожания).

## Данные

- **Fantasy-фид** (открытый, без auth): цены, очки по категориям, % капитана.
- **Jolpica**: календарь+времена сессий, результаты, история с 1950.
- **OpenF1** (18 эндпоинтов): location 3.7Гц, car_data, stints, pit,
  intervals, race_control, team_radio, weather. Live — €9.90/мес; фаза 1 —
  replay-режим бесплатно.
- **FastF1**: телеметрия, стинты, деградация — для превью и разборов.
- **Приватные API Fantasy/Predict** — Phase 0: снять трафик авторизованной
  сессии, замапить (логин-токен, вопросы Predict + коэффициенты, отправка
  ответов, состав/трансферы), написать фетчеры и сабмиттеры кодом с ретраями
  и алертом на 401. Рутины не зависят от MCP-браузера.

## Фазы

- **Phase 0** — разведка API (до 17.07). Блокер автономной отправки.
- **Phase 1** — движок: превью Спа (вт 14.07), пики, разбор; журнал; ingest
  открытых источников. Дебют R10 Спа даже без UI.
- **Phase 2** — приложение по дизайну v1 (Vite + React 19 + Hono + SQLite,
  стек Gaffer), деплой Vercel; replay-центр.
- **Phase 3** — live-тариф OpenF1, лига-аналитика соперников, углубление
  моделей (пит-крю, дождь, SC).

## Real-time / вебсокеты (решение 2026-07-09)

- Phase 1–2: вебсокеты НЕ нужны. Replay = разовая REST-загрузка исторических
  массивов + клиентский курсор времени. Остальные экраны — fetch/refetch.
- Phase 3 (live): источник — MQTT/WS из платной подписки OpenF1 (поллинг не
  вывезет location 3.7Гц × 22 машины при лимите 6 req/s); доставка в браузер —
  SSE (один зритель), мост в Hono-сервере.
- Vercel serverless не держит WS/SSE → live-центр работает от локального
  сервера (гонку смотрим дома); Vercel-витрина = replay + все остальные экраны.
- Слой данных Live-центра — интерфейс `TimingSource` с реализациями
  replay-из-БД / поллинг / WS-мост; экран не знает источника.
