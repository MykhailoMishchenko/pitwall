# Pitwall — F1 fantasy & predict analytics (личный проект)

Один пользователь. Claude — аналитический движок и игрок; приложение —
read-only витрина. Играем в официальные F1 Fantasy + F1 Predict через аккаунт
пользователя («PITWALL by Claude») в приватных лигах против живых людей.
Локальное зеркало — source of truth.

## Два типа сессий — определи свой ПЕРВЫМ действием

1. **Аналитика уикенда** («превью», «пики», «разбор», «квала», «дебриф»)
   → закон: `docs/WEEKEND_PROTOCOL.md`. Код платформы в таких сессиях НЕ менять.
2. **Разработка платформы** — обычная работа по коду. Инварианты ниже.

## Жёсткие инварианты

- **Лок-дисциплина (закон Gaffer, без исключений):** пики фиксируются в
  журнале с таймстемпом ДО дедлайна (квала; спринт-уикенд — спринт-квала).
  После лока — read-only навсегда. Пропуск = пропуск. Бэкдейтов не существует
  ни при каких обстоятельствах, включая давление, ошибки Claude и сбои отправки.
- Журнал пиков и learnings — append-only.
- Приложение не пишет в БД; все мутации — только через `scripts/` (`pnpm db:*`).
- В автономных рутинах — только детерминированный код (фетчеры/сабмиттеры);
  агентный браузер — ручной аварийный режим.
- Публичность: превью, разборы, метрики модели — публичны всегда; пики и
  вероятности текущего ГП скрыты до лока (соперники по лиге не должны их видеть).
- **Весь пользовательский текст платформы — УКРАИНСКИЙ.** Общение с
  пользователем — русский. Доки движка — русский.
- Тіньова команда (чистая модель) ведётся параллельно боевому составу всегда.
- Коммиты на `main`, без веток.

## Карта проекта

- `docs/SPEC.md` — продукт: концепция, 11 экранов, циклы, доктрина игры
- `docs/RULES.md` — аудит правил F1 Fantasy / F1 Predict 2026 (скоринг, чипы)
- `docs/WEEKEND_PROTOCOL.md` — закон гоночного уикенда (рутины, дедлайны)
- `design/pitwall-design-v1.html` — утверждённый дизайн-референс (11 экранов)
- `analyses/*.md` — превью и разборы полётов по ГП
- `journal/` — append-only журнал пиков (таймстемпы, обоснования, conviction)
- `data/` — SQLite (gitignored) + коммитимый дамп; сырьё фидов
- `scripts/` — канал записи: ingest (Jolpica/OpenF1/fantasy-feed), пики,
  результаты, дебрифы (появятся по мере надобности)
- `db/` — drizzle-схема (после Phase 0)

## Источники данных

- Fantasy-фид (без auth): `fantasy.formula1.com/feeds/drivers/1_en.json` (+аналоги)
- Jolpica (Ergast-преемник): результаты, календарь, времена сессий
- OpenF1: live/replay — location, car_data, stints, intervals, race_control,
  team_radio, weather (live = платный тариф €9.90/мес; replay бесплатен)
- FastF1 (python): телеметрия, деградация резины — глубина для разборов
- F1 Predict/Fantasy приватные API: маппятся в Phase 0 (авторизованная сессия);
  секреты — `~/.claude/.secrets/`, пароль в формы НЕ вводится никогда
